HTML

HTML defines the structure and content of the page.

Tags
tag description special attributes self-closing
<html> contains all tags and content - no
<head> contains information for the page - no
<meta> describes metadata for the page; is placed within <head> name, content, charset yes
<link> defines relationship between current document and external resource; is placed within <head> rel, href yes
<title> contains the title for the page; is placed within <head> - no
<body> contains all content that is displayed on the page - no
<main> semantic tag that contains the main content of the page - no
<header> semantic tag that contains content for the header of the page, such as title and nav - no
<footer> semantic tag that contains content for the footer of the page, such as copyright and contact info - no
<nav> semantic tag that contains links for navigation through the site, often organized in <ul> - no
<section> semantic tag that defines a section of content that can be grouped together logically - no
<article> semantic tag that contains a unit of content on a distinct topic, often starting with a heading that describes it - no
<aside> semantic tag that contains content that refers to or adds information to another part of the page, or contains a side navigation - no
<span> non-semantic tag that is used to add specific styles to a small portion of content; should be used rarely and when adding a class doesn't make sense style is often defined inline no
<div> non-semantic tag that is used to section off a piece of content; should be used rarely, choosing an appropriate semantic tag whenever possible; short for divider - no
<h1>, <h2>, <h3>, <h4>, <h5>, <h6> heading tags that define content based on its importance on the page (h1 is most important, h6 is least important); often, but not neccessarily, have sizes that correspond with the heading (h1 is often the largest) - no
<p> paragraph tag that holds text content, often in large amounts - no
<ul>, <ol>, <li> unordered list, ordered list, and list item tags that hold content to be displayed as a list - no
<img> tag to display an image src, alt, width, height yes
